#pragma once : est une directive du préprocesseur utilisée pour empêcher 
que les fichiers en-tête soient inclus plusieurs fois. 
La directive #pragma once , présente dans un fichier, garantit que 
le fichier ne sera pas inclus plusieurs fois dans le projet en cours.
Utilisé dans : tous les headers du projet.


--CPP5--

Classe imbriquée : est une classe déclarée dans une autre classe.
Elle s'utilise avec l'opérateur de résolution de portée (::).
Cela regroupe des types fortement liés à la logique de la classe parente.
Utilisé dans : CPP5/ex00 (exceptions de Bureaucrat).

Exception : est un mécanisme qui signale une erreur à l'exécution avec throw.
Le bloc try encadre le code risqué et catch récupère l'erreur.
L'exception remonte la pile jusqu'au premier gestionnaire compatible.
Utilisé dans : CPP5/ex00, ex01, ex02, ex03.

Exception personnalisée : est une classe d'erreur héritée de std::exception.
Elle redéfinit what() pour fournir un message précis.
Ce type d'exception permet de distinguer clairement les causes d'échec.
Utilisé dans : CPP5/ex00, ex01, ex02, ex03.

Forward declaration : est une déclaration anticipée d'une classe sans son corps.
Elle réduit les dépendances et évite les inclusions circulaires entre headers.
Elle suffit pour déclarer des pointeurs et des références.
Utilisé dans : CPP5/ex01, ex02, ex03.

Template Method : est un patron où la classe de base définit un algorithme.
Les classes dérivées implémentent seulement les étapes spécifiques.
Cela centralise les règles communes et simplifie la maintenance.
Utilisé dans : CPP5/ex02, ex03 (AForm et formulaires dérivés).

Dispatch table : est un tableau associant un nom à une fonction de création.
Elle remplace des if/else répétés par une sélection de fonction.
Ajouter un nouveau type revient à ajouter une seule entrée.
Utilisé dans : CPP5/ex03 (Intern::makeForm).

Factory method statique : est une méthode static qui construit un objet.
Elle retourne souvent un pointeur polymorphe vers la classe de base.
Cette approche uniformise la création d'objets selon un type demandé.
Utilisé dans : CPP5/ex03 (création des formulaires).


--CPP6--

static_cast : est une conversion explicite vérifiée à la compilation.
Il sert aux conversions sûres entre types compatibles.
Il est plus lisible et plus sûr qu'un cast en style C.
Utilisé dans : CPP6/ex00.

std::numeric_limits : est un utilitaire qui donne les bornes d'un type.
Il permet de vérifier min() et max() avant une conversion.
Cela évite les dépassements lors du changement de type.
Utilisé dans : CPP6/ex00.

Classe non-instantiable : est une classe qu'on ne peut pas construire.
Ses constructeurs sont privés et ses méthodes sont souvent static.
Elle sert à regrouper des utilitaires sans état d'objet.
Utilisé dans : CPP6/ex00, ex01.

reinterpret_cast : est une conversion bas niveau d'une représentation binaire.
Il réinterprète l'adresse ou les bits sans vérifier la validité sémantique.
Son usage doit rester limité aux cas techniques comme la sérialisation.
Utilisé dans : CPP6/ex01.

Sérialisation de pointeur : est la conversion Data* vers uintptr_t.
La désérialisation inverse convertit uintptr_t vers Data*.
L'objectif est de préserver la même adresse de mémoire.
Utilisé dans : CPP6/ex01.

dynamic_cast : est une conversion sûre dans une hiérarchie polymorphe.
Avec un pointeur, il retourne NULL en cas d'échec de conversion.
Avec une référence, il lance une exception std::bad_cast.
Utilisé dans : CPP6/ex02.

RTTI : est le mécanisme qui fournit le type réel à l'exécution.
Il fonctionne avec les classes possédant au moins une méthode virtuelle.
dynamic_cast s'appuie sur RTTI pour valider les conversions.
Utilisé dans : CPP6/ex02.


--CPP7--

Template de fonction : est une fonction générique paramétrée par un type T.
Le compilateur génère une version pour chaque type réellement utilisé.
Cette approche évite de dupliquer la même logique.
Utilisé dans : CPP7/ex00, ex01.

Instanciation implicite : est la déduction automatique du type template.
Le compilateur détermine T à partir des arguments passés à l'appel.
La bonne version de la fonction est générée sans annotation supplémentaire.
Utilisé dans : CPP7/ex00, ex01, ex02.

Surcharge template const/non-const : adapte le comportement au niveau de const.
Elle permet de traiter des tableaux modifiables et non modifiables.
Le compilateur choisit automatiquement la surcharge la plus adaptée.
Utilisé dans : CPP7/ex01, ex02.

Template de classe : est une classe générique comme Array<T>.
Chaque type instancié produit une classe distincte à la compilation.
On obtient une structure réutilisable pour plusieurs types.
Utilisé dans : CPP7/ex02.

Surcharge de operator[] : permet l'accès aux éléments d'une classe conteneur.
Elle retourne une référence pour lire ou modifier un élément.
Un contrôle des bornes peut lancer std::out_of_range.
Utilisé dans : CPP7/ex02.

Allocation générique new T[n]() : crée un tableau de type template T.
Le constructeur par défaut de T est appelé pour chaque élément.
Les types primitifs sont initialisés à zéro avec les parenthèses.
Utilisé dans : CPP7/ex02.


--CPP8--

Algorithme STL std::find : recherche une valeur dans un intervalle d'itérateurs.
Il compare chaque élément entre begin() et end().
Si la valeur n'est pas trouvée, il retourne l'itérateur end().
Utilisé dans : CPP8/ex00 (easyfind).

Type dépendant template (typename T::iterator) : indique un type membre d'un paramètre template.
Le mot-clé typename est obligatoire pour lever l'ambiguite lors de la compilation.
Cela permet d'ecrire une fonction generique qui retourne l'itérateur du conteneur recu.
Utilisé dans : CPP8/ex00 (easyfind).

Exception std::runtime_error : signale une erreur d'execution avec un message.
Elle est adaptee aux erreurs logiques detectees au runtime.
Elle est utilisee quand une valeur est absente ou quand le conteneur est dans un etat invalide.
Utilisé dans : CPP8/ex00, ex01.

Conteneur STL std::vector : stocke des elements contigus en memoire.
Il offre un acces rapide et des operations push_back efficaces en moyenne.
Il est adapte pour accumuler des entiers puis appliquer des algorithmes STL.
Utilisé dans : CPP8/ex01 (Span).

Calcul du plus petit ecart (shortest span) : consiste a trier puis comparer les voisins.
Apres tri, l'ecart minimum global se trouve forcement entre deux elements adjacents.
Cette methode evite une comparaison quadratique de toutes les paires.
Utilisé dans : CPP8/ex01 (Span::shortestSpan).

Algorithmes std::min_element et std::max_element : trouvent minimum et maximum d'une plage.
Le plus grand ecart est la difference max - min.
Cette approche donne longest span simplement et efficacement.
Utilisé dans : CPP8/ex01 (Span::longestSpan).

Adaptateur de conteneur std::stack : fournit une interface LIFO au-dessus d'un conteneur sous-jacent.
Il expose push/pop/top/size mais pas d'iteration directe.
Le conteneur interne est accessible dans une classe derivee via le membre protege c.
Utilisé dans : CPP8/ex02 (MutantStack).

Iterabilite par heritage (MutantStack) : derive de std::stack pour exposer des iterateurs.
On definit les types iterator/const_iterator/reverse_iterator depuis container_type.
Les methodes begin/end/rbegin/rend redirigent vers le conteneur sous-jacent c.
Utilisé dans : CPP8/ex02.


--CPP9--

Conteneur associatif STL std::map : stocke des paires cle/valeur triees par cle.
Il garantit l'ordre des cles et permet des recherches efficaces.
Il est adapte pour lier une date (cle) a un taux de change (valeur).
Utilisé dans : CPP9/ex00 (BitcoinExchange).

Recherche de date la plus proche avec lower_bound : retourne le premier element >= cle.
Si la date exacte n'existe pas, on recule d'un iterateur pour recuperer la date precedente.
Cette technique permet d'appliquer le dernier taux connu avant une date donnee.
Utilisé dans : CPP9/ex00/ex02 (BitcoinExchange::findRateForDate).

Validation numerique stricte avec std::strtod + endPtr : verifie qu'une chaine represente exactement un nombre.
La conversion est valide seulement si toute la chaine est consommee (pas de caractere residuel).
Cela evite d'accepter des entrees partielles ou mal formees.
Utilisé dans : CPP9/ex00 (BitcoinExchange::parseValue).

Tokenisation avec std::istringstream : lit une chaine comme un flux d'entree.
Elle permet d'extraire les tokens separes par espaces avec l'operateur >>.
C'est pratique pour evaluer une expression RPN pas a pas.
Utilisé dans : CPP9/ex01 (RPN::evaluate).

Machine a pile pour la notation polonaise inverse (RPN) : applique les operateurs sur les 2 derniers operandes.
Chaque nombre est empile, puis un operateur depile deux valeurs et empile le resultat.
Le calcul est valide si une seule valeur reste en fin d'evaluation.
Utilisé dans : CPP9/ex01.

Algorithme STL std::merge : fusionne deux plages deja triees en une plage triee.
Combine avec une division recursive du tableau, il permet une approche type merge sort.
Cette methode illustre une strategie divide-and-conquer en tri.
Utilisé dans : CPP9/ex02 (PmergeMe::sort).

Ford-Johnson Algorithm (Merge-Insertion Sort) : classe les elements en paires, les trie, puis trie les gagnants.
Les perdants sont ensuite inseres en ordre croissant dans la sequence triee via recherche binaire.
Cet algorithme minimise le nombre de comparaisons par rapport au merge sort classique.
Utilisé dans : CPP9/ex02.

Conteneur STL std::deque : stocke des elements avec acces rapide aux deux extremites.
Contrairement au vector, il permet des insertions/suppressions efficaces au debut et a la fin.
Il est generalement plus gourmand en memoire mais offre une meilleure localite en debut/fin.
Utilisé dans : CPP9/ex02.

Chronomètrage avec std::chrono : mesure le temps d'execution precisement.
std::chrono::high_resolution_clock::now() capture l'instant actuel.
La difference entre deux instants donne la duree en unites personnalisables (microsecondes, nanosecondes, etc).
Utilisé dans : CPP9/ex02 (mesure du temps de tri pour chaque conteneur).

Paire de donnees avec std::pair : groupe deux elements de types potentiellement differents.
Elle offre un moyen simple de manipuler et trier des couples (p.ex. paires et leurs gagnants).
Les paires peuvent etre creees avec std::make_pair ou l'accès par .first et .second.
Utilisé dans : CPP9/ex02 (groupement des paires d'elements avant tri).

Validation de nombre positif stricte : verifie qu'une chaine represente un entier positif.
Elle detecte les nombres negatifs, les caracteres non numeriques et les depassements d'intervalle.
Il est courant d'utiliser std::strtol avec verification du resultat et de endptr.
Utilisé dans : CPP9/ex02 (acceptation seulement d'entiers positifs en argument).
