CPP5 - EXCEPTIONS ET CONCEPTION ORIENTÉE OBJET AVANCÉE
═══════════════════════════════════════════════════════════════════════════════

1. Gestion d'exceptions
   - throw lance une exception et interrompt l'exécution
   - try encadre le code à risque, catch intercepte et gère l'erreur
   - L'exception remonte la pile jusqu'au premier catch compatible
   - Attraper par référence (const) pour éviter la copie

2. Exceptions personnalisées
   - Hériter de std::exception, surcharger what() const throw()
   - Catchable via std::exception& avec message spécifique
   - Une classe par type d'erreur (GradeTooHighException, GradeTooLowException)

3. Classes imbriquées (nested classes)
   - Classe déclarée dans le bloc public/private d'une autre classe
   - Accès via :: (Bureaucrat::GradeTooHighException)
   - Regroupe des types liés sémantiquement à la classe parent

4. Forward declaration
   - Syntaxe: class NomClasse; (sans corps)
   - Évite les dépendances circulaires entre headers
   - Suffit pour déclarer des pointeurs/références, #include va dans le .cpp

5. Couplage faible
   - Collaboration par référence sans connaître les détails internes
   - Déléguer la logique à la classe concernée (responsabilité unique)

6. Pattern Template Method
   - Classe de base définit le squelette, dérivées implémentent les étapes
   - Vérifications communes centralisées, action spécifique déléguée
   - Modification d'une règle = un seul endroit à changer

7. Dispatch table
   - Tableau { nom, fonction } remplaçant if/else en cascade
   - Extensible: ajouter un type = une ligne dans le tableau
   - Structure: { std::string name; AForm* (*create)(const std::string&); }

8. Factory methods statiques
   - Méthode static créant et retournant une instance polymorphe
   - Stockable dans un tableau (impossible avec méthode non-static)
   - Retourne NULL en cas d'échec, appelant responsable du delete

CPP6 - CONVERSIONS DE TYPES (CASTS)
═══════════════════════════════════════════════════════════════════════════════

─── EX00 : Conversion of scalar types ───

1. static_cast<Type>()
   - Conversion explicite type-safe, vérifiée à la compilation
   - Usage: promotions numériques (int→float), downcasts (float→int)
   - Plus sûr que C-casts: (int)variable

2. Classe non-instantiable
   - Constructeurs et destructeur en private
   - Méthodes static uniquement, appel via ClassName::method()
   - Utile pour fonctions utilitaires sans état

3. Détection de type literal
   - Parser format: 'c' (char), 42 (int), 42.0f (float), 42.0 (double)
   - Gérer pseudo-literals: nan, nanf, +inf, -inf, +inff, -inff

4. Conversion string → nombre
   - strtol/strtof/strtod: conversion robuste avec errno (ERANGE pour overflow)
   - Plus fiable que atoi/atof (gestion d'erreurs)

5. Valeurs spéciales IEEE 754
   - NaN (Not a Number): opération invalide (0/0, sqrt(-1))
   - ±inf: overflow ou division par zéro
   - std::isnan(x), std::isinf(x) pour tester avant conversion

6. std::numeric_limits<Type>
   - ::min(), ::max() pour obtenir limites d'un type
   - Vérifier overflow avant conversion

7. Affichage formaté
   - std::fixed + std::setprecision(n): contrôler décimales
   - #include <iomanip>

─── EX01 : Serialization ───

8. reinterpret_cast<Type>()
   - Conversion bas niveau sans vérification, réinterprétation brute des bits
   - DANGEREUX: aucune conversion de valeur
   - Usage: sérialisation, interfaçage C, manipulation d'adresses

9. uintptr_t
   - Type entier assez grand pour stocker une adresse (#include <stdint.h>)
   - Conversion réversible: ptr → uintptr_t → ptr (même adresse)

10. Sérialisation de pointeurs
    - reinterpret_cast<uintptr_t>(ptr): Data* → uintptr_t
    - reinterpret_cast<Data*>(raw): uintptr_t → Data*
    - Vérifie ptr_original == ptr_désérialisé

─── EX02 : Identify real type ───

11. dynamic_cast<Type>()
    - Conversion sûre dans hiérarchie polymorphe (nécessite virtual)
    - Vérifie le type réel à l'exécution (RTTI)
    
    Avec pointeurs:
    - Retourne NULL si échec, DOIT tester != NULL
    
    Avec références:
    - Lance std::bad_cast si échec, DOIT utiliser try/catch

12. Identification de type sans typeinfo
    - Tester chaque classe avec dynamic_cast successivement
    - Alternative à typeid() et <typeinfo>

13. RTTI (Run-Time Type Information)
    - Mécanisme C++ pour identifier le type réel à l'exécution
    - Activé automatiquement si classe a ≥1 méthode virtuelle
    - Utilisé par dynamic_cast
    - Coût: mémoire (v-table) et performance (vérification runtime)
- RTTI : mécanisme runtime pour connaître le type réel (nécessite virtual)

CPP7 - TEMPLATES ET PROGRAMMATION GÉNÉRIQUE
═══════════════════════════════════════════════════════════════════════════════

─── EX00 : Function templates ───

1. Templates de fonctions
   - Syntaxe: template <typename T> Type fonction(T arg)
   - Compilation générant du code pour chaque type utilisé
   - Paramètre de type: T remplacé par le type réel à la compilation
   - Instanciation implicite: swap(a, b) → compilateur déduit T

2. Surcharge implicite par le compilateur
   - Un template = un modèle pour générer plusieurs fonctions
   - swap<int>(), swap<double>(), swap<std::string>() générées automatiquement
   - Chaque instanciation est une fonction entièrement indépendante

3. Non-type template parameters (optionnel)
   - Paramètres constantes en temps de compilation: template <typename T, int N>
   - Utile pour les tableaux fixes: Array<int, 10> crée 10 ints

4. Template specialization
   - Surcharge: template <> void swap<std::string>(...)
   - Spécialiser pour un type spécifique si besoin

─── EX01 : Templates avancés et pointeurs de fonctions ───

5. Templates multiples paramètres
   - template <typename T, typename Function>
   - Permet flexibilité: iter(tab, len, func) applique func à chaque élément

6. Pointeurs de fonctions comme paramètres template
   - Function ft: paramètre template acceptant fonction ou foncteur
   - Appel: ft(add[i]) invoque le callback sur chaque élément
   - Même fonction template avec signatures différentes

7. Overloading de templates
   - Variantes pour const et non-const: iter(T*) et iter(const T*)
   - Compilateur choisit la meilleure surcharge selon le contexte
   - Permet traiter const T* sans modification d'éléments

8. Template d'exception
   - std::out_of_range pour bounds checking
   - throw std::out_of_range("message") lance exception standard

─── EX02 : Template de classe ───

9. Templates de classes
   - Syntaxe: template <typename T> class Array
   - T remplace type d'élément, généré pour chaque utilisation
   - Array<int> et Array<double> sont des classes entièrement différentes

10. Constructeurs en templates
    - Constructeur par défaut: Array() initialise à vide
    - Constructeur avec taille: Array(unsigned int n) alloue n éléments
    - Initialisation des éléments: new T[n]() appelle constructeur de T

11. Copy constructor et operator= en template
    - Deep copy obligatoire: copier chaque élément du tableau
    - operator=: libérer l'ancienne mémoire, allouer nouvelle, copier
    - Vérification this != &other pour éviter auto-destruction

12. Opérateur[] surchargé en template
    - Deux versions: T& operator[]() et const T& operator[]() const
    - Bounds checking: throw std::out_of_range si idx >= size
    - Retour par référence pour permettre modification

13. Destructeur et gestion mémoire
    - delete[] _data libère tableau alloué avec new[]
    - Essentiel: ne pas fuir de mémoire, appel automatique
    - Valide même si _data == nullptr

14. Allocation de tableau générique
    - new T[n](): alloue n éléments de type T
    - Appelle constructeur par défaut de T pour chaque élément
    - () initialise à zéro si T est primitif (int(), double() = 0)

15. Instanciation explicite vs implicite
    - Implicite: Array<int> arr(5); → compilateur génère Array<int>
    - Code généré séparément pour chaque type instancié
    - Augmente taille du binaire vs code générique en runtime
